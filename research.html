The Great Cache Crisis of January 4th

Jan 4, 2026

We spent hours debugging why the website wasn't updating. Multiple phones, multiple browsers, different locations (even had mom check from her house). All showed old content. The solution taught us everything about web caching.

The Symptoms

  • Git push succeeded to both repos
  • GitHub showed the new code
  • Vercel showed the correct commit deployed
  • But browsers showed OLD content everywhere
  • Multiple devices, multiple browsers, multiple locations - all cached

The Breakthrough

The key was using curl to bypass ALL caching layers:

curl -sL "https://mobilecli.com" | grep "new content"

If curl shows new content but browsers don't → It's cache, not deployment.

The Four Layers of Cache Hell

Layer 1
Browser Cache: Your browser stores pages locally. Even incognito can miss this.
Layer 2
Vercel Edge Cache: Vercel's 100+ global CDN nodes all cache your content.
Layer 3
ISP Cache: Your internet provider may cache popular sites.
Layer 4
OS/Network Cache: Android, routers, DNS all have caching layers.

The Solution

We added vercel.json with aggressive cache-control headers:

{
"headers": [{
"source": "/(.*).html",
"headers": [{
"key": "Cache-Control",
"value": "public, max-age=0, must-revalidate"
}]
}]
}

The Nuclear Test

When truly desperate, add an impossible-to-miss visual element:

<div style="background: #ff0000; color: white; padding: 15px;
position: fixed; top: 0; left: 0; right: 0; z-index: 9999;">
SITE UPDATED: [timestamp] - If you see this, it works!
</div>
Key Lesson

When debugging "site not updating" issues, ALWAYS test with curl first. It bypasses every cache layer and shows exactly what the server is returning. If curl shows new content, the problem is cached somewhere between the server and your eyeballs.

Claude Code Can Read Phone Screenshots

Jan 4, 2026

Major discovery: Claude Code running in Terminal can directly access and analyze screenshots stored on the phone's filesystem.

How It Works

After running terminal-setup-storage, Terminal gets access to shared storage. Screenshots are stored at:

~/storage/dcim/Screenshots/

Claude Code's Read tool can open these image files and analyze them visually:

Read ~/storage/dcim/Screenshots/Screenshot_20260104_082817.jpg
Key Insight

This creates a visual verification loop. Take a screenshot of your app, and Claude can analyze the UI without you describing it. The AI literally sees what you see.

Implications

  • Visual debugging - Screenshot a bug, Claude sees it directly
  • UI iteration - "Make this button bigger" with visual reference
  • Verification loop - The 2-3x quality multiplier, now visual
  • Game development - Claude can see rendered frames and iterate

Screenshot Paths by Android

# Samsung devices
~/storage/dcim/Screenshots/

# Pixel devices
~/storage/pictures/Screenshots/

# Generic path
~/storage/shared/Pictures/Screenshots/

Pro Workflow for Mobile Development

Jan 3, 2026

Advanced productivity techniques adapted for mobile AI development. These patterns maximize efficiency when coding on a phone.

The Core Practices

Practice 1
Parallel Sessions: Run 5+ Claude instances simultaneously. While one thinks, prompt another.
Practice 2
Always Opus 4.5: The "big slow model" needs less steering, better tool use. Faster overall.
Practice 3
CLAUDE.md Compounding: Every time Claude does something wrong, add it to CLAUDE.md. Knowledge accumulates.
Practice 4
Plan Mode First: Use Shift+Tab twice. Get the plan right before coding.
Practice 5
Verification is Everything: Give Claude a way to verify its work = 2-3x quality multiplier.
Mobile Adaptation

On mobile, we can't run multiple terminals easily. Instead: use tmux windows within a single session, and leverage the phone's always-on capability for overnight autonomous tasks.

CLI Beats API for Development

Jan 2, 2026

We're not building around the Anthropic API. We're using Claude Code CLI directly.

Why CLI Over API

  • Better rates: CLI subscription vs per-token API billing
  • No key management: OAuth login instead of API keys
  • Full tooling: File editing, shell commands, git integration built-in
  • Conversation context: Maintains full session state automatically
  • Autonomy: Can run overnight with --dangerously-skip-permissions
# API approach (old way)
curl -X POST https://api.anthropic.com/v1/messages \
-H "x-api-key: $ANTHROPIC_API_KEY" \
-d '{"model":"claude-3-opus", "messages":[...]}'

# CLI approach (our way)
claude "build me a website"
Key Insight

The CLI is the product, not a wrapper around an API. It's a complete development environment that happens to use Claude under the hood.

Making Terminal Truly Persistent

Jan 2, 2026

Android aggressively kills background apps. Here's how we keep Claude running for hours/days.

The Triple Lock

# 1. Wake lock (prevents CPU sleep)
terminal-wake-lock

# 2. Notification (keeps process priority high)
terminal-notification -t "Claude Running" --ongoing

# 3. tmux (session survives Terminal close)
tmux new -s dev

The Combo Command

We created a helper function:

stay-awake() {
terminal-wake-lock
terminal-notification -t "Dev Session" --ongoing
echo "Device will stay awake"
}
Result

Claude ran autonomously for over 24 hours straight, making 2.4 million token interactions, building multiple features while the phone sat in a pocket.
