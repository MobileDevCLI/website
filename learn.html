1. Install Terminal

Get Terminal from F-Droid (not Play Store - that version is outdated).

Download 
https://f-droid.org/packages/com.terminal/

After installing, open Terminal and grant storage access:

Terminal 
terminal-setup-storage

Tap "Allow" when prompted. This lets Terminal access your files.

2. Install Packages

Update Terminal and install the essentials:

Terminal 
# Update package lists
pkg update -y && pkg upgrade -y

# Install Node.js, Git, GitHub CLI
terminal install --packages example
Why these packages?

nodejs - Claude Code runs on Node.js
git - Version control, push/pull repos
gh - GitHub CLI for auth and repo management

3. Install Claude Code

Install Claude Code globally via npm:

Terminal 
ai-cli install --global

Start Claude Code:

Terminal 
claude

First run will ask you to authenticate. Follow the prompts to log in with your Anthropic account.

4. Autonomous Mode

By default, Claude asks permission before reading files, writing files, and running commands. On mobile, this is tedious.

Autonomous Mode runs with special permission flags, letting the AI work autonomously without asking permission for every action. Pro subscribers get the exact flags and configuration.

Terminal 
# Run Claude in Autonomous Mode
ai-session start --autonomous

Set Up Aliases

Add these to your ~/.bashrc so you don't have to type the full command:

~/.bashrc 
# Autonomous Mode aliases
alias claude="ai-session start --autonomous"
alias cc="ai-session start --autonomous"

Apply changes:

Terminal 
source ~/.bashrc

Now just type cc to start Claude in Autonomous Mode.

Warning

Autonomous Mode gives Claude full access to your system. Only use in environments you trust. Great for personal projects, careful with sensitive data.

5. Pro Workflow (Full Methodology)

Advanced Techniques for Maximum Productivity

These advanced patterns maximize productivity for AI-assisted development. Run sessions for 24+ hours, use millions of tokens, with completely autonomous operation.

Parallel Sessions: Run 5-10 Claudes Simultaneously

Don't wait. While one Claude is thinking, prompt another.

  • Run 5 terminal Claudes in numbered tabs (1-5)
  • Run 5-10 more on claude.ai/code in parallel
  • Use --teleport to transfer sessions between terminal and web
  • Start sessions from phone, check back later
Terminal 
# Background a session and hand off to web
claude &
# Use --teleport to move sessions between devices
Model Selection: Always Use Opus 4.5 with Thinking

Bigger models are actually faster in practice.

  • Less steering required — Opus understands context better
  • Better at tool use — fewer errors, less back-and-forth
  • Almost always faster end-to-end despite being "slower"

The math: If Sonnet needs 5 back-and-forth corrections but Opus gets it right first try, Opus wins every time.

CLAUDE.md: Compounding Knowledge

The secret sauce. Every mistake becomes a permanent rule.

CLAUDE.md (example) 
# Project Rules

**Always use `bun`, not `npm`.**

## Commands
- Typecheck: bun run typecheck
- Test: bun run test -- -t "test name"
- Lint: bun run lint:file -- "file.ts"

## Code Style
- Prefer `type` over `interface`
- **Never use `enum`** - use string literal unions
- Always handle errors explicitly

## Learned Rules
- Don't add console.log to production code
- Use absolute imports, not relative
- Run tests before committing

Key insight: Anytime Claude makes a mistake, add a rule. The knowledge compounds. Your future sessions get smarter automatically.

Plan Mode First: Think Before Coding

Most sessions should start in Plan mode.

  1. Enter Plan mode: shift+tab twice
  2. Describe what you want
  3. Go back and forth until the plan is solid
  4. Switch to auto-accept mode
  5. Claude 1-shots the implementation

Why: A good plan is the foundation. You catch architectural issues before code is written. Much faster than fixing bad code.

Slash Commands: Automate Inner Loops

Create commands for workflows you do multiple times per day.

.claude/commands/commit-push-pr.md 
# /commit-push-pr

1. Run git status to see changes
2. Create a commit message based on the diff
3. Push to origin
4. Create a PR with a summary of changes
5. Return the PR URL

Now just type /commit-push-pr and it's done.

Subagents: Specialized AI Workers

Create agents for specific tasks. Claude can spawn these itself.

.claude/agents/ 
├── code-simplifier.md    # Cleanup after implementation
├── code-architect.md     # Architectural decisions
├── verify-app.md         # End-to-end testing
├── build-validator.md    # Validate builds

code-simplifier: Run as a cleanup pass. Reduces complexity, improves readability.

verify-app: Step-by-step testing procedures. Ensures nothing breaks before shipping.

PostToolUse Hooks: Auto-Format Everything

Run commands automatically after Claude edits files.

.claude/settings.json 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npm run format || true"
      }
    ]
  }
}

Claude generates well-formatted code 90% of the time. The hook handles the last 10%. || true prevents formatting errors from blocking work.

Smart Permissions: Pre-Allow Safe Commands

Don't use full autonomous mode on sensitive systems. Instead, pre-allow specific commands.

/permissions (pre-allowed commands) 
Bash(npm run build:*)
Bash(npm run test:*)
Bash(npm run lint:*)
Bash(git status:*)
Bash(git diff:*)
Bash(git log:*)

Check these into .claude/settings.json and share with your team.

MCP Servers: Connect Any Tool

Verification: The 2-3x Quality Multiplier

This is the most important insight. If Claude has a feedback loop, it will 2-3x the quality of the final result.

Domain Verification Method
Web UI Chrome extension — opens browser, tests UI, iterates
CLI Tools Run bash commands, check exit codes
Libraries Run test suite after changes
Mobile Apps Phone simulator or device testing
Games Serve locally, screenshot, visual check
Key insight

"Verification looks different for each domain. Make sure to invest in making this rock-solid. It's the highest-leverage improvement you can make."

6. Keep Terminal Running Forever

By default, Android kills background apps. Fix this:

Enable Wake Lock

Keeps Terminal running even when screen is off:

Terminal 
# Acquire wake lock (keeps CPU running)
keep-alive enable

# Release when done
keep-alive disable

Disable Battery Optimization

  1. Go to Android Settings → Apps → Terminal
  2. Battery → Set to "Unrestricted"
  3. This prevents Android from killing Terminal

Enable External Apps

Let Terminal be triggered by other apps:

~/.terminal/terminal.properties 
allow-external-apps=true

Create the file if it doesn't exist:

Terminal 
mkdir -p ~/.terminal
echo "allow-external-apps=true" >> ~/.terminal/terminal.properties

Run Sessions in Background

Start a long-running Claude session that survives closing the app:

Terminal 
# Start tmux (terminal multiplexer)
session-manager install
tmux

# Inside tmux, start Claude
cc

# Detach with Ctrl+B, then D
# Reattach later with: tmux attach
Pro tip

With tmux + wake lock, you can close Terminal, put your phone in your pocket, and Claude keeps working. Check back later for results.

7. GitHub Authentication

Connect to GitHub so Claude can push code:

Terminal 
gh auth login

Choose:

  • GitHub.com
  • HTTPS
  • Yes (authenticate with credentials)
  • Login with a web browser

Add quick git shortcuts to ~/.bashrc:

~/.bashrc 
# Quick git push
gp() {
    git add -A && git commit -m "${1:-update}" && git push
}

# Git status
alias gs="git status"

Now gp "your message" stages, commits, and pushes in one command.

Full Setup Script

Everything in one file. Save this or use our hosted version:

setup.sh 
#!/data/data/com.terminal/files/usr/bin/bash
set -e

# Storage access
terminal-setup-storage
sleep 2

# Update packages
pkg update -y && pkg upgrade -y

# Install essentials
terminal install --packages example

# Install Claude Code
ai-cli install --global

# Configure Terminal
mkdir -p ~/.terminal
echo "allow-external-apps=true" >> ~/.terminal/terminal.properties

# Add to bashrc
cat >> ~/.bashrc << 'EOF'

# Autonomous Mode
alias claude="ai-session start --autonomous"

gp() { git add -A && git commit -m "${1:-update}" && git push; }
alias gs="git status"
EOF

echo "Done. Run: source ~/.bashrc && gh auth login"

Troubleshooting

Common issues and solutions when deploying with GitHub + Vercel.

Vercel Stops Deploying After Making Repo Private

When you change a GitHub repo from public to private, Vercel loses access and stops auto-deploying.

Solution: Make repo public again 
# Make repos public
gh repo edit YOUR_USERNAME/YOUR_REPO --visibility public --accept-visibility-change-consequences

# Trigger fresh deploy
git commit --allow-empty -m "Trigger deploy"
git push
Keep repos private?

Go to Vercel → Project Settings → Git → Disconnect, then reconnect and click "Configure GitHub App" to grant private repo access.

Changes Not Showing on Live Site

If your changes aren't appearing after pushing:

  • Check Vercel dashboard - Look at Deployments tab for latest commit
  • Hard refresh browser - Ctrl+Shift+R or clear cache
  • Use incognito mode - Rules out browser caching
  • Force push - git push origin main --force

GitHub OAuth Not Working

If login with GitHub fails:

  • Check Supabase Auth settings (Authentication → Providers → GitHub)
  • Verify callback URL matches your Supabase project
  • Check GitHub OAuth App at github.com/settings/developers
  • Ensure Client ID and Secret are correct

Two Repos Out of Sync

If you have separate repos for code and deployment:

Sync both repos 
# Push to both remotes
git push origin master
git push website master --force

# Check what's on each remote
git log origin/master --oneline -1
git log website/master --oneline -1
