1. Install Termux

Get Termux from F-Droid (not Play Store - that version is outdated).

Download 
https://f-droid.org/packages/com.termux/

After installing, open Termux and grant storage access:

Terminal 
termux-setup-storage

Tap "Allow" when prompted. This lets Termux access your files.

2. Install Packages

Update Termux and install the essentials:

Terminal 
# Update package lists
pkg update -y && pkg upgrade -y

# Install Node.js, Git, GitHub CLI
pkg install -y nodejs git gh
Why these packages?

nodejs - Claude Code runs on Node.js
git - Version control, push/pull repos
gh - GitHub CLI for auth and repo management

3. Install Claude Code

Install Claude Code globally via npm:

Terminal 
npm install -g @anthropic-ai/claude-code

Start Claude Code:

Terminal 
claude

First run will ask you to authenticate. Follow the prompts to log in with your Anthropic account.

4. Boris Mode (Autonomous)

By default, Claude asks permission before reading files, writing files, and running commands. On mobile, this is tedious.

Boris Mode (named after Boris Cherny, creator of Claude Code) runs with --dangerously-skip-permissions, letting Claude work autonomously.

Terminal 
# Run Claude in Boris Mode
claude --dangerously-skip-permissions

Set Up Aliases

Add these to your ~/.bashrc so you don't have to type the full command:

~/.bashrc 
# Boris Mode aliases
alias claude="claude --dangerously-skip-permissions"
alias cc="claude --dangerously-skip-permissions"

Apply changes:

Terminal 
source ~/.bashrc

Now just type cc to start Claude in Boris Mode.

Warning

Boris Mode gives Claude full access to your system. Only use in environments you trust. Great for personal projects, careful with sensitive data.

5. Keep Termux Running Forever

By default, Android kills background apps. Fix this:

Enable Wake Lock

Keeps Termux running even when screen is off:

Terminal 
# Acquire wake lock (keeps CPU running)
termux-wake-lock

# Release when done
termux-wake-unlock

Disable Battery Optimization

  1. Go to Android Settings → Apps → Termux
  2. Battery → Set to "Unrestricted"
  3. This prevents Android from killing Termux

Enable External Apps

Let Termux be triggered by other apps:

~/.termux/termux.properties 
allow-external-apps=true

Create the file if it doesn't exist:

Terminal 
mkdir -p ~/.termux
echo "allow-external-apps=true" >> ~/.termux/termux.properties

Run Sessions in Background

Start a long-running Claude session that survives closing the app:

Terminal 
# Start tmux (terminal multiplexer)
pkg install -y tmux
tmux

# Inside tmux, start Claude
cc

# Detach with Ctrl+B, then D
# Reattach later with: tmux attach
Pro tip

With tmux + wake lock, you can close Termux, put your phone in your pocket, and Claude keeps working. Check back later for results.

6. GitHub Authentication

Connect to GitHub so Claude can push code:

Terminal 
gh auth login

Choose:

  • GitHub.com
  • HTTPS
  • Yes (authenticate with credentials)
  • Login with a web browser

Add quick git shortcuts to ~/.bashrc:

~/.bashrc 
# Quick git push
gp() {
    git add -A && git commit -m "${1:-update}" && git push
}

# Git status
alias gs="git status"

Now gp "your message" stages, commits, and pushes in one command.

7. The Workflow

Here's how you actually use this:

Terminal 
# 1. Start Claude in Boris Mode
cc

# 2. Tell Claude what you want
# "Build me a landing page with a contact form"
# "Fix the bug in auth.js"
# "Add dark mode to the app"

# 3. Claude reads files, writes code, runs commands
# You watch or do something else

# 4. Push when ready
gp "Add landing page"

That's it. Describe what you want, let Claude work, push the result.

9. Advanced: Hooks & Agents

For power users who want more control.

Hooks

Claude Code supports hooks that run before/after tool calls. Configure in .claude/settings.json:

.claude/settings.json 
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "command": "npm run format || true"
      }
    ]
  }
}

This auto-formats code after every write/edit.

Subagents

Create specialized agents in .claude/agents/:

.claude/agents/code-reviewer.md 
# Code Reviewer Agent

Review the code for:
- Security vulnerabilities
- Performance issues
- Code style consistency
- Missing error handling

Provide specific, actionable feedback.

Slash Commands

Create custom commands in .claude/commands/:

.claude/commands/deploy.md 
# /deploy

1. Run tests: npm test
2. Build: npm run build
3. Commit with message "Deploy: [current date]"
4. Push to main
5. Verify deployment succeeded

Run with /deploy in Claude.

CLAUDE.md

Create a CLAUDE.md in your project root. Claude reads this for context:

CLAUDE.md 
# Project Guidelines

## Commands
- Build: npm run build
- Test: npm test
- Lint: npm run lint

## Rules
- Use TypeScript
- No console.log in production
- Always handle errors

## Notes
- API keys are in .env
- Deploy triggers on push to main
Key insight

Every time Claude makes a mistake, add a rule to CLAUDE.md. The knowledge compounds. Future sessions get smarter.

Full Setup Script

Everything in one file. Save this or use our hosted version:

setup.sh 
#!/data/data/com.termux/files/usr/bin/bash
set -e

# Storage access
termux-setup-storage
sleep 2

# Update packages
pkg update -y && pkg upgrade -y

# Install essentials
pkg install -y nodejs git gh

# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Configure Termux
mkdir -p ~/.termux
echo "allow-external-apps=true" >> ~/.termux/termux.properties

# Add to bashrc
cat >> ~/.bashrc << 'EOF'

# Boris Mode
alias claude="claude --dangerously-skip-permissions"
alias cc="claude --dangerously-skip-permissions"

# Git shortcuts
gp() { git add -A && git commit -m "${1:-update}" && git push; }
alias gs="git status"
EOF

echo "Done. Run: source ~/.bashrc && gh auth login"
