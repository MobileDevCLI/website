Complete Setup Guide

The exact steps to set up your mobile development environment. Real commands, no placeholders.

The Stack

Here's exactly what we use:

  • Termux ‚Äî Linux terminal emulator for Android (F-Droid version only)
  • Termux:API ‚Äî Hardware access (camera, notifications, clipboard)
  • Claude Code ‚Äî Anthropic's AI coding CLI (npm package)
  • tmux ‚Äî Terminal multiplexer for persistent sessions
  • Node.js LTS ‚Äî JavaScript runtime
  • Git + GitHub CLI ‚Äî Version control and GitHub integration

Step-by-Step Installation

1

Install Termux

Download Termux from F-Droid (NOT Google Play ‚Äî that version is outdated):

https://f-droid.org/packages/com.termux/

Also install Termux:API from F-Droid for hardware access.

2

Update and Install Dependencies

Bash 
# Update packages
pkg update -y && pkg upgrade -y

# Install core dependencies
pkg install nodejs-lts git gh openssh -y

# Install Termux:API package (after installing Termux:API app)
pkg install termux-api -y
3

Install Claude Code

Bash 
# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Verify installation
claude --version

You'll need an Anthropic account. Claude Code will prompt you to authenticate via browser on first run.

4

Setup GitHub Authentication

Bash 
# Login to GitHub CLI
gh auth login

# Choose:
# - GitHub.com
# - HTTPS
# - Login with a web browser
# - Follow the prompts
5

Install tmux for Persistent Sessions

Bash 
# Install tmux
pkg install tmux -y

# Create a new session named 'dev'
tmux new -s dev

# To detach: press Ctrl+B then D
# To reattach later:
tmux attach -t dev
6

Grant Storage Access

Bash 
# Grant access to phone storage
termux-setup-storage

# Now you can access:
~/storage/shared/       # Internal storage
~/storage/dcim/         # Camera/Screenshots
~/storage/downloads/    # Downloads folder

Commands & API Reference

All the real commands. Copy, paste, run.

Running Claude Code

Standard Mode

Bash 
# Run Claude in your project directory
cd ~/your-project
claude

Autonomous Mode (The Power Move)

Bash 
# Run Claude without permission prompts
claude --dangerously-skip-permissions

# This allows Claude to:
# - Read/write files without asking
# - Run bash commands without asking
# - Work autonomously for hours

Use Autonomous Mode Carefully

Only use --dangerously-skip-permissions in projects where you trust what Claude might do. It can read, write, and execute anything.

24+ Hour Sessions

The magic combo for sessions that run for days:

Bash ‚Äî The Full Setup 
# 1. Prevent CPU sleep
termux-wake-lock

# 2. Keep Termux priority (prevents Android killing it)
termux-notification -t "Dev Session" --ongoing -c "Claude running..."

# 3. Start tmux session (survives Termux close)
tmux new -s dev

# 4. Run Claude autonomously
claude --dangerously-skip-permissions

# Now you can:
# - Close the Termux app (session keeps running)
# - Lock your phone (session keeps running)
# - Come back hours later and it's still going

Reattach to Running Session

Bash 
# Open Termux and reattach
tmux attach -t dev

# List all sessions
tmux ls

Termux:API ‚Äî Hardware Access

Full hardware access from the terminal:

Bash ‚Äî Hardware Commands

CLAUDE.md Template

## Quick Context
What this project is and what you're building.

## Tech Stack
- List your technologies
- Any specific versions

## Important Rules
- Never do X
- Always do Y
- Preferred patterns


Research Lab

Technical discoveries from building AI development tools on mobile. Updated as we learn.

Capability

Claude Code Can Read Phone Screenshots

After running termux-setup-storage, Termux gets access to shared storage. Claude Code's Read tool can open and analyze screenshots:

Bash ‚Äî Screenshot Access 
# Grant storage access first
termux-setup-storage

# Screenshot locations:
~/storage/dcim/Screenshots/           # Samsung
~/storage/pictures/Screenshots/       # Pixel
~/storage/shared/Pictures/Screenshots/ # Generic

# Claude can read these directly
# Just use: Read ~/storage/dcim/Screenshots/filename.jpg

Use cases: Visual debugging (screenshot a bug, Claude sees it), UI iteration ("make this button bigger" with visual reference), game development (Claude can see rendered frames).

Workflow

The Verification Loop (2-3x Quality Multiplier)

The single most important pattern: Give Claude a way to verify its work. If it can test and iterate, quality jumps 2-3x.

Bash ‚Äî Verification Examples 
# Web development - serve and screenshot
npx http-server -p 8080 &
termux-open-url "http://localhost:8080"
# Take screenshot, Claude analyzes it

# CLI tools - run and check output
node script.js
echo $?  # Claude checks exit code

# Tests - run test suite
npm test
Integration

Multi-Layer Caching (The Great Cache Crisis)

When debugging "site not updating" issues, there are FOUR caching layers that can serve stale content:

  • Browser Cache ‚Äî Your browser stores pages locally
  • Vercel Edge Cache ‚Äî Vercel's global CDN caches at edge nodes
  • ISP Cache ‚Äî Your internet provider may cache
  • OS/Network Cache ‚Äî Device-level caching
Bash ‚Äî Bypass All Caching 
# This bypasses ALL caching and shows what server actually has
curl -sL "https://yoursite.com" | grep "some text"

# If curl shows new content but browser doesn't = cache problem

Troubleshooting

Common issues and how to fix them.

Claude Code Not Installing?

Bash 
# Make sure Node is updated
pkg upgrade nodejs-lts

# Clear npm cache
npm cache clean --force

# Try installing again
npm install -g @anthropic-ai/claude-code

Session Killed by Android?

Bash 
# Make sure you have:
termux-wake-lock                              # Prevents CPU sleep
termux-notification -t "Dev" --ongoing        # Keeps priority

# Also check Android settings:
# Settings > Apps > Termux > Battery > Unrestricted

Storage Access Issues?

Bash 
# Grant storage permission
termux-setup-storage

# Then access via:
~/storage/shared/       # Internal storage
~/storage/dcim/         # Camera photos
~/storage/downloads/    # Downloads

GitHub Authentication Failed?

Bash 
# Re-authenticate
gh auth logout
gh auth login

# If that fails, try with token:
gh auth login --with-token < token.txt

tmux Session Lost?

Bash 
# List all sessions
tmux ls

# If no sessions, the phone may have rebooted
# Start fresh:
tmux new -s dev

# Prevent future loss - use persistent notification:
termux-notification -t "Dev Session" --ongoing -c "Active"

Claude Not Responding?

Bash 
# Check internet connection
ping -c 3 google.com

# Check if Claude is running
ps aux | grep claude

# Kill and restart if frozen
pkill -f claude
claude

Site Changes Not Showing?

Bash ‚Äî The Definitive Test 
# Bypass ALL caching
curl -sL "https://yoursite.com" | grep "unique text"

# If curl shows new content but browser doesn't:
# It's a cache problem, not a deployment problem

# Clear Vercel cache by redeploying:
git commit --allow-empty -m "Force redeploy"
git push
