Chapter 1

The Starting Point: A Blank Terminal

It started with nothing. A fresh Android phone. A blank Termux terminal. No code, no website, no accounts, no brand. Just an idea: prove that you can build a complete web presence entirely from a phone using AI.

The thesis was simple but bold:

The command line is the great equalizer. On CLI, a $200 Android phone has the same capabilities as a $3000 MacBook Pro. Add AI, and the phone becomes a development powerhouse.

What we set out to build:

  • A complete brand identity (MobileDevCLI)
  • A professional website with multiple pages
  • GitHub organization with repositories
  • Social media presence (Twitter, YouTube)
  • Automated deployment pipeline
  • Installation scripts for others to follow
  • Legal pages, documentation, roadmaps

All from a phone. All in one session. All autonomous - meaning I could give commands and turn my screen off while the AI worked.

Chapter 2

Setting Up the Environment

Step 1: Install Termux

Termux is a terminal emulator that runs a full Linux environment on Android. It's the foundation of everything.

Download from F-Droid (NOT Play Store) 
https://f-droid.org/packages/com.termux/

The Play Store version is outdated. F-Droid has the maintained version with all packages working.

Step 2: Initial Packages

Three commands to get a working development environment:

Terminal 
# Update packages
pkg update && pkg upgrade -y

# Install essentials
pkg install nodejs git gh -y

# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

That's it. Node.js for running JavaScript, Git for version control, GitHub CLI for authentication, and Claude Code - the AI that would do the heavy lifting.

Step 3: Authenticate Everything

Terminal 
# Login to GitHub
gh auth login

# Login to Claude Code (uses Anthropic API key or OAuth)
claude

# Follow the prompts...

The GitHub CLI walks you through browser-based OAuth. Claude Code connects to Anthropic's servers. Both remember your credentials.

Chapter 3

Unleashing Boris Mode

This is where everything changes. Normal Claude Code asks permission for every file write, every command execution. On mobile, typing "yes" repeatedly is painful. Enter Boris Mode.

Boris Mode is named after Boris Cherny, the creator of Claude Code. It runs with --dangerously-skip-permissions, enabling fully autonomous operation.
~/.bashrc 
# Add this alias
alias cc="claude --dangerously-skip-permissions"

# Reload
source ~/.bashrc

# Now just type:
cc

With Boris Mode enabled:

  • Claude reads any file without asking
  • Claude writes/creates files without prompts
  • Claude runs shell commands autonomously
  • Claude commits and pushes to GitHub
  • You can turn off your screen and come back later

This transforms the experience. Instead of constant back-and-forth, you describe what you want, and Claude builds it. You verify when it's done.

Chapter 5

The Matrix Redesign

The initial design was functional but boring. I asked for something iconic:

"I want bleeding Matrix code falling in the hero section. Make it feel like entering the Matrix."

Claude wrote a canvas-based animation with falling Japanese characters and ASCII symbols. Green on black. The signature look was born.

Matrix Animation (simplified) 
const chars = 'アイウエオカキクケコ0123456789';
const drops = Array(columns).fill(1);

function drawMatrix() {
    ctx.fillStyle = 'rgba(0, 0, 0, 0.05)';
    ctx.fillRect(0, 0, canvas.width, canvas.height);
    ctx.fillStyle = '#00ff41';

    for (let i = 0; i < drops.length; i++) {
        const char = chars[Math.floor(Math.random() * chars.length)];
        ctx.fillText(char, i * fontSize, drops[i] * fontSize);
        if (drops[i] * fontSize > canvas.height) drops[i] = 0;
        drops[i]++;
    }
}

setInterval(drawMatrix, 50);

Design decisions made during iteration:

  • Color: #00ff41 (Matrix green) on #000000 (pure black)
  • Font: Monospace for everything code-related
  • Mobile-first: Tab navigation for phones
  • No frameworks: Pure HTML/CSS/JS for simplicity

Chapter 7

The Mobile Fix

Testing on the phone revealed issues. In portrait mode, text was getting cut off on the sides. The code blocks overflowed. Claude fixed it with responsive CSS:

Responsive fixes 
@media (max-width: 768px) {
    .container { padding: 0 16px; }
    .hero-code {
        overflow-x: auto;
        word-break: break-all;
    }
    pre code {
        white-space: pre;
        -webkit-overflow-scrolling: touch;
    }
}

@media (max-width: 400px) {
    .container { padding: 0 12px; }
    .hero h1 { font-size: 1.6rem; }
}

Every page got the treatment: proper container padding, horizontal scrolling for code blocks, smaller fonts on small screens. Now it works in both portrait and landscape.

Chapter 9

Version Control & Safety

With a working website, we needed insurance against breaking changes. Claude set up a proper git workflow:

Creating a stable tag 
# Tag the current stable state
git tag -a v1.0.0 -m "Initial stable release"
git push origin v1.0.0

# Now if something breaks, rollback:
git checkout v1.0.0

The branching strategy:

  • main - Production (auto-deploys to Vercel)
  • develop - Staging/testing
  • feature/* - New features in isolation
  • hotfix/* - Emergency fixes

Rule: Create a tag before any major change. Authentication, payments, redesigns - anything you might want to undo gets tagged first.

Chapter 11

What This Proves

This project is a proof of concept for a new way of building software:

  1. Mobile is not a limitation. With CLI tools, your phone is as capable as any computer.
  2. AI autonomy is transformative. Boris Mode eliminates the friction of constant confirmation prompts.
  3. Verification beats supervision. Let AI work, check results when done. Much faster than watching every step.
  4. Documentation compounds. CLAUDE.md and PROJECT_HISTORY.md make future sessions smarter.
  5. Portability wins. Code from anywhere - commute, coffee shop, bed. Your phone is always with you.

The Workflow

The Pattern 
1. DESCRIBE what you want
   "Build me a landing page with Matrix falling code"

2. LET CLAUDE WORK
   Turn off screen if you want. Claude reads, writes, commits.

3. VERIFY the output
   Check the result. Provide feedback.

4. ITERATE
   "Make the text bigger" / "Add mobile tabs"

5. DOCUMENT
   Update PROJECT_HISTORY.md, create version tags
