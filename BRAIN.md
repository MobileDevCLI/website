## CONTEXT BONDING STRATEGY (How Claude Remembers)

### The Problem: Context Loss During Compaction
When conversations get long, they get "compacted" (summarized). This loses context.
The user noticed: "every once in a while you have to recompile/compact your conversation... there's a lot of context that you've lost after you compact."

### The Solution: Multiple Memory Layers

Claude's memory works in layers. Use ALL of them:

**Layer 1: Conversation History (Cloud)**
- Anthropic stores conversation history
- Just telling Claude something makes it remember
- Example: "You are TEST CLAUDE on the test phone" → Claude remembers instantly
- This is WHY the two-phone workflow works so well

**Layer 2: Documentation Files (CLAUDE.md, BRAIN.md)**
- Survives conversation compaction
- Read at start of each session
- Contains project context, patterns, rules
- Updated by both human AND AI

**Layer 3: Knowledge Base (~/knowledge/)**
- ERRORS.md, PATTERNS.md, NEVER_DO.md
- 5-second additions with scripts
- Persists across ALL sessions forever

**Layer 4: Project Files**
- Comments in code
- README files
- Version history in commits
- The codebase IS documentation

### How to Strengthen the Bond

1. **At session start:**
   - Claude reads CLAUDE.md automatically
   - Claude reads BRAIN.md for inventor context
   - Claude checks knowledge base

2. **During session:**
   - Document discoveries immediately
   - Add to knowledge base (5 seconds!)
   - Update CLAUDE.md with new patterns
   - Commit with descriptive messages

3. **At session end:**
   - Ensure all discoveries are documented
   - Update version numbers
   - Push to GitHub (optional backup)

4. **Cross-session continuity:**
   - Start new session with "resume where we left off"
   - Reference specific files: "we were working on MainActivity.kt"
   - Claude's cloud history fills in gaps

### Why "Tell Claude = Claude Remembers"

The user discovered: when you tell Claude something in a conversation, it becomes part of Claude's understanding for future conversations with you.

```
Session 1: "This phone is TEST CLAUDE"
Session 2: Claude already knows it's TEST CLAUDE (from history)
```

This is NOT magic - it's Anthropic's conversation history system.

**How to leverage this:**
