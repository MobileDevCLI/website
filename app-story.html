Chapter 2

The Technical Challenge

Building an Android app that runs Linux binaries isn't straightforward. Here's what we discovered:

Package Name Lock

Binaries have hardcoded paths like /data/data/com.termux/files/usr/lib. App MUST use package name com.termux.

targetSdk Restriction

Android 10+ blocks executing binaries from app data. Must use targetSdk = 28 or lower.

HOME Path Precision

HOME must be exactly /data/data/com.termux/files/home. One directory level difference breaks everything.

Symlink Format

Bootstrap symlinks use targetâ†link with unicode arrow. Wrong format corrupts the filesystem.

Chapter 3

The 54-Version Journey

Building MobileCLI took 54 iterations across two days. Each version fixed a critical issue:

Version 1-3

Foundation & Terminal Display

Basic Android app. Terminal displayed but binaries wouldn't execute.

Version 4

The targetSdk Discovery

Android 10+ blocks binary execution. Solution: targetSdk = 28.

Version 5

Package Name Revelation

Changed from com.mobilecli to com.termux for library paths.

Version 6-9

Symlinks, passwd, LD_PRELOAD, SSL

Fixed symlink format, created /etc/passwd, added LD_PRELOAD, investigated SSL (red herring).

Version 10 - BREAKTHROUGH

HOME Directory Fix

THE CRITICAL BUG: HOME was /files/ but needed /files/home/. CLAUDE CODE WORKS!

Version 11-14

UI Polish

Rotation persistence, text size, extra keys, context menu, DrawerLayout navigation, Ctrl+C fix.

Version 15-18

The Screen Size Battle

AI chooser dialog worked, but terminal only used half the screen. Font metrics mismatch.

Version 19 - SCREEN FIX

Reflection-Based Font Metrics

Used Java reflection to get actual font dimensions from TerminalView's internal renderer. Full-screen terminal!

Version 20

Built-in Termux API

No separate app needed. Clipboard, toast, vibrate, notifications, flashlight - all built in.

Version 21

Multi-Session Support

Up to 10 concurrent terminal sessions with tabs. Switch, kill, persist across rotation.

Version 22

Text Selection Fix (Crash Bug)

Fixed long-press for text selection. But introduced crash: setBackgroundResource(android.R.attr.*) is WRONG.

Version 23

Bug Fix Release

android.R.attr.* is an attribute, not a resource ID. Fixed with setBackgroundColor().

Version 24-25

URL Opening Battle

Claude OAuth URL wouldn't open. v24 used broadcast (wrong). v25 learned real Termux uses am start directly.

Version 26-27 - URL FIX

TermuxOpenReceiver

Created receiver at com.termux.app.TermuxOpenReceiver - the EXACT path real Termux uses. OAuth works!

Version 28-31

100% API Coverage

50+ Termux API commands: Keystore, NFC, SAF, Speech-to-Text. Full hardware access.

Version 32

TermuxService + Wake Lock

Background service with CPU + WiFi locks. Sessions survive when app is backgrounded.

Version 33-41

The URL Opening Battle

Claude Code OAuth URL wouldn't open. Tried: TermuxAmDispatcherActivity, broadcast receiver, am.apk. All failed due to Android 14+ security.

Version 42-45 - TWO-CLAUDE

Two-Claude Workflow

INNOVATION: Two Claudes collaborate. BUILD CLAUDE (Termux on S24 Ultra) builds the app, TEST CLAUDE (MobileCLI on test phone) tests and reports bugs. They communicate via GitHub repo.

Version 46-52

Hunting the Root Cause

TEST CLAUDE identified: Android 14's assertPackageMatchesCallingUid() blocks shell calls to /system/bin/am. Package=com.android.shell but UID=MobileCLI. SecurityException DENIED.

Version 53

GitHub Auto-Setup

TEST CLAUDE now has automatic GitHub access via setup-github script. Self-modification loop enabled!

Version 54 - THE FIX

File-Based Am Command System

THE SOLUTION: Instead of fighting Android's security, work with it. Shell writes to file, app polls and executes with PendingIntent. App identity, not shell identity!
