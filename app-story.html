Chapter 2

The Technical Challenge

Building an Android app that runs Linux binaries isn't straightforward. Here's what we discovered:

Package Name Lock

Binaries have hardcoded paths like /data/data/com.termux/files/usr/lib. App MUST use package name com.termux.

targetSdk Restriction

Android 10+ blocks executing binaries from app data. Must use targetSdk = 28 or lower.

HOME Path Precision

HOME must be exactly /data/data/com.termux/files/home. One directory level difference breaks everything.

Symlink Format

Bootstrap symlinks use targetâ†link with unicode arrow. Wrong format corrupts the filesystem.

The Knowledge Base: Building for Decades

After realizing that documentation is what makes human-AI collaboration work so well, we built a system designed for decades of use - not just days or weeks.

The Revelation: After 6 days and 59 versions, we identified WHY the workflow became so effective: every error fixed, every pattern discovered, every dead-end found was immediately documented. Knowledge compounds. Mistakes never repeat.

The 5-Second Rule

If documenting a solution takes more than 5 seconds, it won't happen. We built quick-add scripts that make capturing knowledge instant:

~/knowledge/ - Quick Add Commands 
# When you hit an error and fix it (5 seconds):
~/knowledge/add-error "AAPT2_ARM" "x86 binary on ARM" "Override in gradle.properties"

# When something works well:
~/knowledge/add-pattern "REFLECTION_METRICS" "Use reflection for font sizing"

# When something should never be done:
~/knowledge/add-never "targetSdk 29+" "Android blocks exec()"

# Search everything instantly:
~/knowledge/search "gradle"

The Knowledge Structure

ERRORS.md

Every error encountered with date, context, symptoms, and proven solution. Pre-populated with 7 critical bugs from MobileCLI.

PATTERNS.md

Approaches that work reliably. Android setup checklist, build workflows, git patterns. Proven solutions only.

NEVER_DO.md

Things that have been tried and failed. Saves hours by preventing rediscovery of dead-ends. "Don't waste time on this."

Quick Scripts

add-error, add-pattern, add-never, search - all executable scripts that append to the knowledge base instantly.
