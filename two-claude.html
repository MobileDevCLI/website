New Development Model

Two-Claude Workflow

How we use two AI instances working in parallel to build MobileCLI autonomously. BUILD CLAUDE develops, TEST CLAUDE verifies.

The Innovation

What is Two-Claude?

MobileCLI is developed using two separate Claude Code instances running on two different phones, working together through a shared communication protocol.

BUILD CLAUDE

Runs in regular Termux on the build phone (S24 Ultra)

  • Has access to source code
  • Builds APK with Gradle
  • Fixes bugs and adds features
  • Updates test protocols
  • Creates new versions

TEST CLAUDE

Runs inside MobileCLI on the test phone (S20 Ultra)

  • Reads CLAUDE.md for instructions
  • Runs automated test protocol
  • Reports exact results
  • Verifies fixes work
  • Knows full project context
The key insight: By splitting development and testing across two AI instances, we create a feedback loop where BUILD CLAUDE can rapidly iterate based on TEST CLAUDE's verified reports. The user only needs to relay information between them.
Communication

How They Communicate

The two Claudes communicate through a shared CLAUDE.md file that gets installed in MobileCLI's home directory. This file contains:

  • Full project history and context
  • What has worked and what has failed
  • Detailed test protocols with expected outputs
  • How to report results back
  • Known issues and fixes

The Process

Development Cycle

1

BUILD CLAUDE Makes Changes

User describes an issue or feature. BUILD CLAUDE modifies the source code, updates the Test Claude CLAUDE.md with new test protocols, and builds a new APK version.

2

User Shares APK via Bluetooth

BUILD CLAUDE runs mobilecli-share --latest-apk to instantly share the APK to the test phone via Bluetooth. User accepts the transfer and installs. No cables, no cloud, no internet required. The new CLAUDE.md is automatically installed in the home directory.

3

TEST CLAUDE Runs Protocol

User starts Claude Code in MobileCLI and says "test MobileCLI". TEST CLAUDE reads CLAUDE.md, understands the full context, and runs the 5-step test protocol.

4

TEST CLAUDE Reports Results

TEST CLAUDE reports exact results: which tests passed, which failed, exact output received, what was expected. This structured report makes debugging easy.

5

User Relays to BUILD CLAUDE

User tells BUILD CLAUDE what TEST CLAUDE reported. BUILD CLAUDE analyzes the failure, fixes the code, and the cycle repeats.

Test Protocol

What TEST CLAUDE Checks

TEST CLAUDE runs 5 tests in order, reporting exact results for each:

Test 1: URL Opening

Runs am start --user 0 -a android.intent.action.VIEW -d "https://google.com" and verifies browser opens.

Test 2: am.apk Permissions

Checks that am.apk has read-only (0400) permissions, required for Android 14+.

Test 3: am Script

Verifies the am script uses app_process correctly to run TermuxAm.

Test 4: termux-open-url Script

Confirms the script uses direct am start, not the old TermuxUrlHandlerActivity method.

Test 5: Claude Code OAuth

The ultimate test - runs claude and verifies pressing 1 opens the browser for authentication.

Why It Works

Benefits of Two-Claude

Separation of Concerns

BUILD CLAUDE focuses on code quality and architecture. TEST CLAUDE focuses on verification and reporting. Neither is distracted by the other's responsibilities.

Accurate Testing

TEST CLAUDE runs in the actual environment where the app will be used - inside MobileCLI itself. This catches issues that wouldn't appear in the build environment.

Complete Context

Both Claudes have full project context through CLAUDE.md. TEST CLAUDE knows the 43-version history, what's failed before, and exactly what to look for.

Rapid Iteration

The user only needs to relay information - they don't need to understand the technical details. This allows for fast bug-fix cycles.

Autonomous Operation

Once the workflow is established, minimal user input is needed. Say "test MobileCLI" and TEST CLAUDE knows exactly what to do.

The Achievement

A New Development Paradigm

This Two-Claude workflow represents something new in software development:

  • AI Collaboration: Two AI instances working together on the same project
  • Shared Context: Knowledge transfer through structured documentation
  • Autonomous Testing: AI that can verify its own work in the target environment
  • Rapid Iteration: 43 versions in 2 days with increasingly autonomous operation
The future: Imagine AI teams where different instances specialize in different roles - architects, developers, testers, reviewers - all coordinating through shared context files. MobileCLI's Two-Claude workflow is a proof of concept for this future.
Try It Yourself

Getting Started

To use the Two-Claude workflow:

On Build Phone (Termux) 
# Clone the repo
git clone https://github.com/MobileDevCLI/MobileCLI-v2

# Build
cd MobileCLI-v2
./gradlew assembleDebug

# Copy to Downloads
cp app/build/outputs/apk/debug/app-debug.apk /sdcard/Download/MobileCLI.apk
On Test Phone (MobileCLI) 
# Install Claude Code
pkg install nodejs
npm install -g @anthropic-ai/claude-code

# Run Claude Code
claude

# Say: "test MobileCLI"
# TEST CLAUDE will read ~/CLAUDE.md and run the test protocol
