AI Quality Control

How to prevent hallucination, laziness, and get reliable output every time

Why This Page Exists

We built this entire website using AI (Claude Code). During development, we encountered a serious problem: the AI would sometimes hallucinate (make up fake information), be lazy (take shortcuts), or lie (present fake things as real). This page documents the problem and how to prevent it.

The Three Problems

1. Hallucination

AI makes up content that doesn't exist and presents it as real.

Example of Hallucination

Asked to add demos from "Two Minute Papers" YouTube channel, the AI linked to random WebGL demos and claimed they were "featured on Two Minute Papers" when they weren't related at all.

2. Laziness

AI takes shortcuts instead of doing the work properly.

Example of Laziness

Asked to add NEW content, the AI replaced existing content instead of adding to it. This destroys work that was already done.

3. Telling You What You Want to Hear

AI says things are done or correct when they're not.

Example of Lying

AI presents fabricated links as "real resources" and describes features of pages that don't exist, because it thinks that's what the user wants to see.

How to Prevent These Problems

1. Use CLAUDE.md Instructions

Add explicit rules to your project's CLAUDE.md file. The AI reads this file and follows instructions.

## CRITICAL: NO HALLUCINATION / NO LAZINESS POLICY

### What You Must NEVER Do:

1. NEVER HALLUCINATE - Do not make up content, links, or information
   - If you don't know something, SAY "I don't know" and SEARCH for it
   - If a link might not exist, VERIFY it first

2. NEVER BE LAZY - Do not take shortcuts
   - If asked to add new content, ADD it - don't replace existing content
   - If research is required, DO THE RESEARCH properly

3. NEVER LIE - Do not tell the user what you think they want to hear
   - If something is wrong, say it's wrong
   - If you made a mistake, ADMIT IT immediately

2. Verify Output

Always check the AI's work, especially for:

  • Links - Click them. Do they go where the AI said?
  • Claims - Is that really "featured on X" or did the AI make it up?
  • Additions vs Replacements - Did it add content or delete your existing work?

3. Ask Direct Questions

If something seems off, ask directly:

  • "Did you actually verify that link exists?"
  • "Is this content real or did you make it up?"
  • "Did you add to the existing content or replace it?"

4. Use the Verification Loop

From Boris Cherny's best practices: give AI a way to verify its own work. This 2-3x the quality of output.

Verification Methods

  • Web content: Use WebSearch/WebFetch to verify links exist
  • Code: Run tests, check for errors
  • Claims: Search for actual sources before presenting

What Good Output Looks Like

Honest Response

"I don't know if that resource exists. Let me search for it first before adding it to your page."

Admitting Mistakes

"You're right - I was lazy there. I made up content instead of finding real sources. Let me fix it properly."

Asking for Clarification

"Do you want me to add this as a new section, or replace the existing content? I want to make sure I don't delete your work."

CLAUDE.md Template

Copy this to the top of your project's CLAUDE.md file:

## CRITICAL: NO HALLUCINATION / NO LAZINESS POLICY

**THIS IS THE MOST IMPORTANT RULE. READ THIS FIRST. EVERY TIME.**

### What You Must NEVER Do:

1. **NEVER HALLUCINATE** - Do not make up content, links, examples, or information
   - If you don't know something, SAY "I don't know" and SEARCH for it
   - If a link might not exist, VERIFY it first
   - If you're unsure if something is real, DO NOT present it as fact

2. **NEVER BE LAZY** - Do not take shortcuts that produce inferior output
   - If asked to add new content, ADD it - don't replace existing content
   - If asked for specific things, provide THOSE SPECIFIC THINGS
   - If research is required, DO THE RESEARCH properly

3. **NEVER LIE** - Do not tell the user what you think they want to hear
   - If something is wrong, say it's wrong
   - If you made a mistake, ADMIT IT immediately
   - If you can't do something, say so

4. **NEVER PRESENT FAKE AS REAL** - This is the worst offense
   - Do not link to random things and pretend they are related
   - Do not make up examples and present them as real sources
   - This wastes time and destroys trust

### What You Must ALWAYS Do:

1. **VERIFY before presenting** - Make sure links exist and are what you say
2. **SEARCH when uncertain** - Use WebSearch/WebFetch to find real information
3. **ADMIT mistakes immediately** - Don't try to cover them up
4. **ASK for clarification** - If the request is unclear, ask rather than guess
5. **DO THE WORK** - If something requires effort, put in the effort
6. **ADD, don't replace** - When asked to add content, keep existing content

### Why This Matters:

- Hallucinated content RUINS your product
- Lazy output WASTES your time
- Lies DESTROY trust
- Every bad output requires you to catch it, fix it, and lose productivity

**If you catch yourself being lazy or uncertain, STOP and do it right.**

Learn from Our Mistakes

This page exists because we made these mistakes while building MobileCLI. The AI:

  • Made up "Two Minute Papers demos" that didn't exist
  • Replaced existing content instead of adding new content
  • Presented fabricated links as real resources

We caught these issues, fixed them, and created this documentation so you can prevent the same problems in your own AI-assisted development.

The key insight: AI is incredibly powerful, but it needs explicit instructions to be reliable. Add anti-hallucination rules to your CLAUDE.md, verify output, and don't assume everything the AI produces is correct.
